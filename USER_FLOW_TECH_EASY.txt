■ 사용자 여정 → “그때 쓰는 기술” (순서대로)
--------------------------------------------------------------------------------

  (1) 앱에 들어온다
      · 화면: React + React Router (주소로 페이지 전환)
      · 스타일: Tailwind, 컴포넌트는 Radix(shadcn 스타일)

  (2) 로그인한다 (선택)
      · Supabase Auth + Google OAuth
      · 로그인 상태: AuthContext 가 앱 전체에 전달

  (3) 지도 탭에서 내 주변 상장사를 본다
      · 지도: Leaflet / react-leaflet
      · 데이터: Supabase에 저장된 주변 기업 + 필요 시 /api/companies/nearby (크롤링 발표때는 질문에 대답에만 사용)
      · 시세 숫자: /api/quotes (서버가 Yahoo 쪽 시세 로직 사용) (상동 크롤링함)

  (4) 종목을 눌러 시트를 연다
      · 시세 다시 요청 가능: /api/quotes
      · “기업 알아보기” 챗: /api/chat + 종목 정보(stockAssist)
          서버가 네이버 뉴스·웹 스니펫을 붙인 뒤 OpenAI에 질문 전달

  (5) 하단 가운데 버튼으로 키키(글로벌 챗)를 연다 (로그인 필요)
      · 일반 대화: /api/chat → OpenAI
      · “오늘의 주식 퀴즈”: 지도에 쌓인 종목 스냅샷 → /api/quiz/hybrid
          OpenAI가 문제 JSON 생성 → 앱이 1~4로 채점·캐시 지급 캐시는 문제를 출제한 llm이 난이도 1~10 지정 후 난이도에 따라 캐시(1~10)지급
      · 대화 저장: 글로벌 챗은 브라우저 localStorage 에도 저장

  (6) 걷기 / 보유 / 프로필 탭
      공통: 세 화면 모두 `useUserData` 훅이 같은 상태(걸음·캐시·닉네임·보유·스크랩)를 묶어서 씁니다.

      · 걷기 탭 (/walk)
          - 화면에 보이는 것: 오늘 걸음/일일 목표 걸음, 달성률, 캐시(원), 최근 7일 막대·차트 느낌의 주간 걸음.
          - React 상태: `walk`(오늘 걸음·목표·캐시·걸음당 캐시 비율), `weeklySteps` 등.
          - 로그인 + Supabase 연결 시 DB:
              `user_profiles` … 닉네임, 캐시 잔액(`cash_balance`), 걸음당 적립(`cash_per_step`), 프로필 기본 목표(`goal_steps`).
              `user_walk_daily` … 날짜별(`walk_date`) 오늘 걸음(`steps`)과 그날 목표(`goal_steps`). 첫 로그인 시 행이 없으면 만들어 넣음.
          - 주간 그래프: 같은 `user_walk_daily`에서 최근 7일치를 읽어 요일별 걸음을 채움.
          - 스마트폰(Capacitor)에서는 네이티브 걸음 플러그인이 걸음을 올리면 `addSteps`가 호출되어, 화면 즉시 반영 + 위 테이블들에 upsert/update.
          - 비로그인: `MOCK_USER_WALK` 등 가짜 숫자로 UI만 동작(저장은 기기 메모리 수준).

      · 보유 탭 (/holdings)
          - 보유 종목 목록: 로그인 시 Supabase `user_holdings`(티커, 종목명, 수량, 평단, 현재가)에서 읽음. 매수 시 `buyStock`이 캐시 차감·보유 행 갱신 후 DB에도 반영.
          - 스크랩한 종목: DB가 아니라 브라우저 `localStorage` 키 `kiki_scraps_v1:<사용자id>` 에 JSON으로 저장(기기·브라우저마다 다를 수 있음).

      · 프로필 탭 (/profile)
          - 상단: `user_profiles`에서 온 닉네임, 아래 카드에 보유 캐시(`walk.cashBalance`)와 보유 종목 평가금(같은 훅의 `holdings`로 합산).
          - 키움·워키포인트 연동·보안·앱 설정 등 메뉴는 현재 UI/안내 위주(실제 증권 API 연동은 별도 구현 범위).

  (7) 스마트폰 앱으로 쓴다
      · Capacitor 로 웹 빌드 결과를 네이티브 앱으로 감쌈
      · API 주소는 VITE_CHAT_API_ORIGIN 등으로 고정 도메인 지정 가능

■ 서버(API)가 하는 일 — 쉬운 표
--------------------------------------------------------------------------------
  주소 (사용자는 이렇게만 보면 됨)     실제 동작 (개발자용)
  --------------------------------    --------------------------------------------
  /api/chat                           OpenAI 채팅 (종목이면 뉴스·검색 보강 후)
  /api/quotes                         시세 조회 (Yahoo 경유 로직)
  /api/quiz/hybrid                    퀴즈 문제 생성 (OpenAI 1번 + 검증)
