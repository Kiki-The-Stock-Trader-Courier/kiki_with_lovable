■ 한눈에 (30초)
--------------------------------------------------------------------------------
  사용자가 보는 것     →  React 웹앱 (Vite로 빌드)
  로그인               →  Supabase (구글 계정)
  지도·걸음·캐시·챗   →  브라우저 + Supabase DB + Vercel 서버(API)
  AI 답변·퀴즈 출제    →  서버가 OpenAI(gpt-4o-mini) 호출 (키는 서버에만)


■ 사용자 여정 → “그때 쓰는 기술” (순서대로)
--------------------------------------------------------------------------------

  (1) 앱에 들어온다
      · 화면: React + React Router (주소로 페이지 전환)
      · 스타일: Tailwind, 컴포넌트는 Radix(shadcn 스타일)

  (2) 로그인한다 (선택)
      · Supabase Auth + Google OAuth
      · 로그인 상태: AuthContext 가 앱 전체에 전달

  (3) 지도 탭에서 내 주변 상장사를 본다
      · 지도: Leaflet / react-leaflet
      · 데이터: Supabase에 저장된 주변 기업 + 필요 시 /api/companies/nearby
      · 시세 숫자: /api/quotes (서버가 Yahoo 쪽 시세 로직 사용)

  (4) 종목을 눌러 시트를 연다
      · 시세 다시 요청 가능: /api/quotes
      · “기업 알아보기” 챗: /api/chat + 종목 정보(stockAssist)
          서버가 네이버 뉴스·웹 스니펫을 붙인 뒤 OpenAI에 질문 전달

  (5) 하단 가운데 버튼으로 키키(글로벌 챗)를 연다 (로그인 필요)
      · 일반 대화: /api/chat → OpenAI
      · “오늘의 주식 퀴즈”: 지도에 쌓인 종목 스냅샷 → /api/quiz/hybrid
          OpenAI가 문제 JSON 생성 → 앱이 1~4로 채점·캐시 지급
      · 대화 저장: 글로벌 챗은 브라우저 localStorage 에도 저장

  (6) 걷기 / 보유 / 프로필 탭
      · 걸음·캐시·목표: 화면 상태 + (로그인 시) Supabase user_profiles 등

  (7) 스마트폰 앱으로 쓴다
      · Capacitor 로 웹 빌드 결과를 네이티브 앱으로 감쌈
      · API 주소는 VITE_CHAT_API_ORIGIN 등으로 고정 도메인 지정 가능

■ 서버(API)가 하는 일 — 쉬운 표
--------------------------------------------------------------------------------
  주소 (사용자는 이렇게만 보면 됨)     실제 동작 (개발자용)
  --------------------------------    --------------------------------------------
  /api/chat                           OpenAI 채팅 (종목이면 뉴스·검색 보강 후)
  /api/quotes                         시세 조회 (Yahoo 경유 로직)
  /api/quiz/hybrid                    퀴즈 문제 생성 (OpenAI 1번 + 검증)
  /api/companies/nearby               주변 상장사 목록 (Supabase 등)
  /api/companies/sync                 운영용 데이터 동기화 (별도 토큰 등)

  배포 시 위 주소들은 모두 api/gw.ts 한 덩어리로 모였다가 내부에서 나뉨
  (Vercel 무료 플랜 함수 개수 제한 때문).
